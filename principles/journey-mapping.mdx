A journey map is a visualization of the process a person goes through to accomplish a goal. It helps you identify where your documentation supports users and where it leaves them without help.

## Five components of a journey map

| Component | What it is |
|---|---|
| **Actor** | The persona or user who experiences the journey. Use one point of view per map. |
| **Scenario and expectations** | The situation the map addresses, tied to the actor's goal. |
| **Journey phases** | High-level stages that organize everything else. |
| **Actions, mindsets, and emotions** | What users do, think, and feel at each stage. Emotions are plotted as a single line showing the highs and lows across phases. |
| **Opportunities** | Insights that answer what needs to be done, who owns the change, and how to measure improvement. |

## Journey phase examples

Different products have different journey phases. These are common patterns.

| Product type | Phases |
|---|---|
| Developer tool | Discover, evaluate, integrate, use, troubleshoot |
| E-commerce | Discover, try, buy, use, seek support |
| Enterprise software | Engage, educate, research, evaluate, justify, adopt |

## Standard journey map format

- **Top:** specific user, specific scenario, corresponding goals
- **Middle:** high-level phases with user actions, thoughts, and emotions
- **Bottom:** takeaways including opportunities, insights, and ownership

## Applying journey mapping to documentation

Before writing a new documentation section, map the user journey for that feature. Ask:
- What does the user want to accomplish?
- What do they know before they arrive at this page?
- What emotions are they experiencing at each stage?
- Where are the gaps between what they need and what the documentation currently provides?

The opportunities section of the map becomes your documentation backlog.

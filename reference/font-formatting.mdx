## Google Docs formatting

Use these settings for all documentation deliverables produced in Google Docs.

| Element | Specification |
|---|---|
| Font | Avenir |
| Heading size | 13pt |
| Normal text | 11pt |

## Markdown formatting conventions

These conventions apply to all Markdown and MDX files.

| Element | Convention |
|---|---|
| Bold | `**like this**`. Use `**`, not `__`. |
| Italics | `_like this_`. Use `_`, not `*`. |
| Inline code | `` `like this` `` |
| Code blocks | Triple backticks with language specified |
| Headings | `#` for H1, `##` for H2, `###` for H3 |

## Why underscores for italics

Use underscores for italics so they're visually distinct from bold in source. This makes it easy to tell apart `_italic_` from `**bold**` when reading raw Markdown.

## Heading case

Sentence case for all headings in all contexts. No title case.

```text
Right: Configure alert sensitivity
Wrong: Configure Alert Sensitivity
```

## Spacing rules

- One blank line between paragraphs
- No extra empty lines between sections
- One blank line before and after code blocks
- No trailing spaces at the end of lines

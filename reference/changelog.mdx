## Initial release

### Site

- Published the Hackmamba Documentation Style Guide as a Mintlify site with Hackmamba brand colors
- Single Docs tab with eight sidebar groups ordered to match the technical writer user journey
- Changelog tab

### Getting started

- **Introduction:** two user modes — new writers learning the guide and experienced writers using it as a reference
- **Six documentation standards:** the test for each standard, tool-focused vs user-focused comparison table
- **Quality rubric and AI guardrails:** full six-criterion weighted scoring table and six enforced guardrails

### Writing rules

- **Voice, tense, and person:** active voice, second person, present tense, may vs might, contractions, hidden verbs, plural antecedents, Microsoft Writing Style Guide key principles
- **Sentence and paragraph rules:** 25-word limit, if/then construction, capitalize after colon, avoid "where" to connect clauses, em-dash guidance, abbreviations, numbers and dates, nested clause example
- **Headings:** sentence case, hierarchy, naming conventions by content type, sidebar naming, one grammatical pattern per level
- **Lists and steps:** bullet vs numbered lists, period consistency, mixed-structure items, system responses are not steps
- **Code:** language after every fence, no dollar signs, ALL CAPS placeholders, comment out expected output, text before every code block
- **UI text and bold:** bold for UI elements, select not click, enter not add, disambiguation for repeated labels, error messages, italics
- **Links and signposting:** inline linking, descriptive link text, when "see" is acceptable, transitions between sections
- **Notes, warnings, and tables:** all four callout types demonstrated, table formatting rules
- **Banned words and phrases:** 11 banned words, 9 banned phrases, 7 mechanical transitions, 5 sparingly-used words

### Content types

- **Diataxis framework:** five types including Get started, how to choose, Divio system rules, Pixar formula
- **Tutorials:** required six-section structure, step formatting, troubleshooting section with error examples
- **How-to guides:** required structure, introduction guidance, prerequisites format
- **API reference:** seven required sections per endpoint, complete worked example
- **Troubleshooting:** structure by symptom, three-part entry format, escalation path, document limitations inline

### Principles

- **User-focused writing:** tool-focused vs user-focused, organize by user intent, what developers actually need
- **Cognitive load:** three types, ten techniques as H3 sections, caveman technique
- **Developer mindset:** developers scan not read, friction, match workflow sequence, show don't tell
- **Information architecture:** four components, IA rules, information fragmentation, sidebar naming
- **Journey mapping:** five components including emotions line, journey phase examples
- **Storytelling:** Pixar framework, user-story driven documentation, storytelling structure, voice and tone, Microsoft voice principles

### API documentation

- **REST concepts:** five HTTP methods, key concepts, URL naming rules
- **Best practices:** hello world principle, OpenAPI Specification, organizing API docs, testing before publishing
- **Code examples:** complete examples, inputs and outputs together, multi-language support

### Tooling

- **Vale and cSpell:** configuration, GitHub Actions integration, full quality workflow
- **GitHub Actions:** core concepts, workflow YAML, pre-built actions, secrets management
- **Mintlify:** docs.json configuration, component usage rules, migration checklist
- **Versioning and Docusaurus:** Read the Docs rules, versioning strategy, redirects, Docusaurus setup

### Collaboration

- **Client engagement process:** four-stage engagement model, audit classification framework
- **SME collaboration:** eight principles, source credibility by role, interview technique
- **Docs as code:** core practices, five-step workflow, tools
- **Technical reviews:** rules for reviewers, doc caretaker, how writers should prepare

### Reference

- **Editing checklist:** six sections with checkbox format for every item
- **Seven action model:** all seven actions with descriptions and metrics, doc sandwich
- **Taxonomies:** advantages over indexes, five facets, tagging in practice
- **Font and formatting:** Google Docs specs, Markdown conventions, spacing rules

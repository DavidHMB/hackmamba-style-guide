## Sentence case

Use sentence case for all headings (H1 through H6). No title case.

| Right | Wrong |
|---|---|
| Connect the device to the network | Connect The Device To The Network |
| Configure motion detection alerts | Configure Motion Detection Alerts |
| Set up user permissions | Set Up User Permissions |

## Heading hierarchy

Every page has one H1 that states the topic clearly. Use H2 and H3 to break content into scannable sections. Never skip heading levels. Start with H2 on a page after the H1.

<Note>
Never stack headings. Every heading must have at least one paragraph beneath it before the next heading appears. Never use bold text as a heading.
</Note>

## Description paragraphs

Every heading must be followed by a description paragraph before any list, table, or code block appears. This gives the reader context before they scan the list.

## Naming conventions by content type

Use consistent naming patterns based on content type.

| Content type | Pattern | Examples |
|---|---|---|
| How-to guides | Bare infinitive | "Connect the device to the network", "Configure alert sensitivity" |
| Tutorials | Bare infinitive | "Log your first request", "Build your first pipeline" |
| Get started | Bare infinitive | "Get started" is the only permitted exception |
| Concepts | Noun phrase | "Architecture overview", "Alert types" |
| Reference | Noun phrase | "API reference", "Keyboard shortcuts" |

## One grammatical pattern per level

Keep one grammatical pattern within the same navigation level or documentation set. Avoid mixing imperative verbs with noun phrases at the same heading level. Mixed patterns make scanning slower and the structure feel inconsistent.

## Sidebar naming

Sidebar labels are part of the navigation system. Keep them user-facing, not org-chart-based.

**Use:** Dashboard, Projects, Billing, Team, Settings

**Avoid:** General, Resources, Miscellaneous, Admin Stuff, Other

The label should make sense even when seen out of context, because users often land on a page from search or a shared link.

## Minimise parentheses

Avoid parentheses in headings. If the content in the parentheses matters, integrate it into the heading directly. If it doesn't matter, remove it.
